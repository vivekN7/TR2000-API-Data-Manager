Key Database Design Decisions

Temporal Data Management

Every table includes EXTRACTION_DATE, IS_CURRENT, VALID_FROM, and VALID_TO fields for historical tracking

Composite Primary Keys

Combination of business keys + extraction timestamp ensures uniqueness across time

Change Detection

HASH_VALUE field for efficient change detection using SCD Type 2 pattern

Audit Fields

USER_NAME, USER_ENTRY_TIME, USER_PROTECTED from API headers preserved

ETL Metadata

ETL_RUN_ID links all records to specific ETL execution for traceability

Multi-Client Ready

CLIENT_CODE field prepared for future multi-client support

ETL Process Flow

1. Initialization

Start ETL run, create run ID, initialize logging

2. Extract Phase

Call API endpoints in parallel batches, handle pagination

3. Transform Phase

Validate data, standardize formats, apply business rules

4. Load Phase

Bulk insert to staging tables, update SCD records

5. Validation

Run quality checks, reconciliation, generate reports

6. Finalization

Update control tables, send notifications, cleanup
